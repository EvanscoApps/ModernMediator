# ModernMediator.AspNetCore

ASP.NET Core integration for [ModernMediator](https://www.nuget.org/packages/ModernMediator). Provides compile-time Minimal API endpoint generation from request handlers decorated with the `[Endpoint]` attribute, plus an `IHttpContextAccessor`-backed `ICurrentUserAccessor` for handlers that need authenticated user context.

## Installation

```bash
dotnet add package ModernMediator.AspNetCore
```

The package transitively includes [ModernMediator](https://www.nuget.org/packages/ModernMediator) and [ModernMediator.AspNetCore.Generators](https://www.nuget.org/packages/ModernMediator.AspNetCore.Generators). You do not need to install those separately.

## Setup

In `Program.cs`:

```csharp
using ModernMediator;

var builder = WebApplication.CreateBuilder(args);

builder.Services.AddModernMediator(config =>
{
    config.RegisterServicesFromAssemblyContaining<Program>();
});

builder.Services.AddModernMediatorAspNetCore();

var app = builder.Build();
app.MapMediatorEndpoints();
app.Run();
```

`AddModernMediatorAspNetCore()` registers `IHttpContextAccessor`, which `HttpContextCurrentUserAccessor` depends on.

`MapMediatorEndpoints()` is generated by `ModernMediator.AspNetCore.Generators` at compile time. It registers every `[Endpoint]`-decorated request handler as a Minimal API route.

## Defining endpoints

Decorate request types with `[Endpoint]`. The constructor takes the route first and the HTTP method second (defaults to `"POST"`):

```csharp
using ModernMediator;
using ModernMediator.AspNetCore;

[Endpoint("/orders/{id:int}", "GET")]
public sealed record GetOrderQuery(int Id) : IRequest<OrderResult>;

public sealed class GetOrderHandler : IRequestHandler<GetOrderQuery, OrderResult>
{
    public Task<OrderResult> Handle(GetOrderQuery query, CancellationToken cancellationToken)
    {
        // ...
    }
}
```

Valid HTTP methods: `"GET"`, `"POST"`, `"PUT"`, `"DELETE"`, `"PATCH"`. `GET` endpoints bind route and query parameters into the request type via `[AsParameters]`; non-`GET` endpoints bind the request body via `[FromBody]`. The generator dispatches each request through `IMediator.Send`.

See the [core ModernMediator README](https://github.com/evanscoapps/ModernMediator) for the full attribute reference and parameter binding rules.

## Current user access

`HttpContextCurrentUserAccessor` exposes the authenticated user's identity to handlers via the `ICurrentUserAccessor` interface (defined in the core `ModernMediator` package). It surfaces two pre-extracted values:

- `UserId` — the `ClaimTypes.NameIdentifier` claim
- `UserName` — `IIdentity.Name`

Both return `null` when no HTTP context is available or the request is unauthenticated.

`HttpContextCurrentUserAccessor` is registered automatically when you opt into the audit pipeline with `UseHttpContextIdentity()`:

```csharp
builder.Services.AddModernMediator(cfg => cfg
    .AddAudit<MyAuditWriter>(o => o.UseHttpContextIdentity(builder.Services)));
```

If you need `ICurrentUserAccessor` outside the audit pipeline, register it explicitly:

```csharp
builder.Services.AddSingleton<ICurrentUserAccessor, HttpContextCurrentUserAccessor>();
```

Then inject it into a handler:

```csharp
public sealed class GetMyOrdersHandler : IRequestHandler<GetMyOrdersQuery, IReadOnlyList<OrderResult>>
{
    private readonly ICurrentUserAccessor _currentUser;

    public GetMyOrdersHandler(ICurrentUserAccessor currentUser)
    {
        _currentUser = currentUser;
    }

    public Task<IReadOnlyList<OrderResult>> Handle(GetMyOrdersQuery query, CancellationToken cancellationToken)
    {
        var userId = _currentUser.UserId;
        // ...
    }
}
```
